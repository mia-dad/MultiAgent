AI 软件工程团队编排系统

MultiAgent 是一个面向未来的软件开发方式而设计的 AI 多智能体编排系统。
它通过定义不同的「软件工程角色」，让 AI 能够以分布式方式执行真实的工程流程，例如：

项目管理

系统分析

架构设计

开发编码

测试验证

部署运维

MultiAgent 的愿景是：
让每一个角色拥有自己的上下文、会话、记忆与能力，通过协作工作流形成完整的软件工程生产线。

🔥 为什么要做 MultiAgent？

当前 AI 编程的最大瓶颈是：

模型无法准确理解用户意图，也无法在复杂流程中保持一致性。

MultiAgent 的解决方案是：

✔ 每个 AI 角色拥有独立上下文（独立大脑）

保证不同角色的推理互不干扰，同时可选择性共享信息。

✔ 分布式架构

支持跨机器部署，让不同角色的上下文可无限扩展。

✔ 基于真实开发流程的工作流

实现一个真正的软件工程团队，而不是单一的大模型对话。

🎯 第阶段目标（已启动）

实现一个最核心的能力：远程 API 控制本地 Terminal 会话

包括：

在 Mac / Linux 上使用 PTY 保持真实 shell

在 Windows 上使用 PowerShell / CMD 管道保持会话

每次对话可绑定同一个 Terminal 会话（上下文保持）

通过 HTTP API 驱动终端操作，供远程 AI 或服务调用

对应的服务：
MultiAgent Terminal Service

它实现了：

POST /session/new — 创建终端会话

POST /session/exec — 执行命令（带上下文）

GET /session/output — 读取会话输出

这是整个 MultiAgent 系统的基础模块，未来所有 AI 角色都会依赖它来：

在本地写代码

执行构建命令

运行测试

部署应用

管理多项目环境

🚀 项目愿景（规划中）

角色系统（Agents）
自定义 PM / 架构师 / 开发 / 测试 / 运维角色能力与上下文。

任务编排引擎（Workflow）
角色之间可按真实流程协作。

分布式执行（Distributed Agents）
不同角色部署在不同机器或容器中。

统一上下文总线（Context Bus）
控制不同角色的上下文共享范围。

基于多轮推理的 AI 规划器（Planner）
管理整体项目目标。
